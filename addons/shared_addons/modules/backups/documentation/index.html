Introduction

Thank you for purchasing PyroBackup! PyroBackup is a powerful yet simple PyroCMS module which helps you and your users to quickly and easily setup and run backups of all your important data. With PyroBackup, you can give your backups (Presets) names and descriptions, pick certain database tables and easily swap the destination of your backups. PyroBackup currently supports the backup via/to the following methods:
  • Email
  • Amazon S3
  • Direct Download
  • Dropbox
To watch and overview of PyroBackup's features, please see the videos.

Requirements

In order to run PyroBackup, you will require the following:
  • Installation of PyroCMS v2.x
  • A copy of PyroBackup, available from the PyroCMS Addons Store.

Installation

To install PyroBackup, please follow the follow steps.
  1. With your copy of PyroBackup, ensure that the Zip file from the Addons store is named 'Backups.zip'
  2. Login with your administrator account on your PyroCMS installation
  3. From the Admin panel, select "Addons" from the top menu
  4. Click on the "+ Upload" button in the top right.
  5. Select the Backups.zip file from your hard drive and then press "Upload"
  6. Your module should now be installed.
Now PyroBackup is installed, you can access it from under the "Content" dropdown menu at the top of the page.

Upgrading

To upgrade the module, copy and replace all of the existing module files with those provided as part of this package. Next, visit your PyroCMS instllation, and browse to the Addons page in the Admin section. If there are no conflicts of versions, an Upgrade button should be visible for the module. Press this button to complete the updgrade.

Terminology

Here is some basic terminology that is used in the PyroBackup module.

Preset
A Preset is a saved configuration for a backup job. A Preset consists of a name, description, list of tables to backup and an optional account.

Account
An Account stores information about a particular backup method. The backup methods that are currently supported include Email, Dropbox and Amazon S3. A Preset can be linked to an Account, so that backups can be pushed automatically to these services.

Getting Started




Lets run through a quick tutorial which will show you how to create a Preset and generate a Cron Job for this.
  1. From the menu at the top of the page, under the Content section, select PyroBackup
  2. In the shortcuts bar, click on "Add Preset"
  3. Give the Preset a name to identify it from other Presets
  4. Optionally give the Preset a description for your own personal reference
  5. Pick which tables should be included
    1. Selecting "All Tables" will result in all tables in the DB at backup runtime to be included in the backup. As a result, if new tables exist at backup runtime, then these new tables will also be included in the backup.
    2. Selecting "Specific Tables" result in a list of available tables being shown. If any of these tables are not available at backup runtime, then they will not be included. If new tables are added, they will not be automatically added.
    3. Selecting "Tables With Prefix" will allow you to select all of the tables that have a specific prefix. This comes in handy if you are running multiple PyroCMS installations from the same database. Once you've specified your prefix (or multiple by comma-seperating them), PyroBackup will check each time the backup is run which tables have the prefix. This is feature is also useful if you do not want to explictely specify which tables to backup, whilst pulling them all.
  6. Pick the Backup Method
    1. Selecting "No Account" will mean that this Preset can only be run automatically and not part of a Cron Job
    2. If you select an option under "Add Account" you will be required to enter the required account details for specified acocunt. For example if you select Amazon S3, you will be required to specify the Access Key, Secret Key, Bucket and optionally a path.
    3. If you have previously made an Account or Preset with a new account, they will also be displayed. Selecting one of these will allow you to link more than one Preset to an Account.
  7. Click the "Save" button.
Once you have saved the new Preset, the module will then check what you have entered to ensure that it is all in the correct format that we require. If you have entered the credentials for a new account, we will also validate this account to ensure that we can connect to it. If any errors occur, you will be returned to the re-populated form and a message will be displayed alerting you to the error.

Once the form has validated successfully, your Preset will be saved and you will be shown the overview page for the Preset. On this page you can view and edit all of the information relating the Preset and the Account related (if applicable).

If you wish for your Preset to be run via a Cron Job, we provide you with a Cron Job Generator. The generator allows you to speciy the particular intevals at which you want the cron job to run, and it will generate the relevant row for you to add into your Crontab file.

The Cron Job Generator provides you with two different methods of execution for the Crontab, which are via cURL and Wget. In order for the Backup script to be accessible to the crontab, each backup much specify both the unique ID of the Preset and text string, as demonstrated below.

http://mysite.com/backups/run/3/qQNL2iFdm9

The unique code attached at the end of this URL is automatically generated by PyroBackups when the Preset is created. If this string is not specified, then the backup will not start. This is designed so that your Presets cannot be run without knowing the key.

Viewing A Preset's Information

To view a Preset, click on the "Presets" link in the shortcut bar and click the "View" button next to the Preset that you would like to view the information of. From this page, you can see all of the information that is related to the Preset.

Running / Downloading a Preset
On this page, two buttons that you are shown are labelled "Run Preset Now" and "Download". The "Run Preset Now" button will mean that this Preset will be run (like you would with a Cron Job) and the backup will go to the account that you have specified.
The "Download" button however, will result in the backup being run, but instead of the backup going to a specified account, the backup will be downloaded in your browser.

Creating, Modifying & Deleting Accounts

Apart from the ability to create an account whilst making a Preset, we also provide you with the ability to create and edit accounts in a seperatley. To view all of the Accounts, click on the "Accounts" link in the shortcuts bar.

Viewing Presets Linked To An Account
To view any Presets which may be linked to an account, simply click on the account, and the list of Presets will be displayed below.

Adding An Account
To add an Account, simply click on the "Add Account" link in the shortcuts bar. You will be presented with be preseted with all the required fields.

Editing An Account
From the main Accounts page, click on the "Edit" button next to the Account that you would like to edit.

Deleting An Account
To delete an Account, click the "Delete" button next to the Acccount that you would like to delete. If a Preset is linked to the account, the Preset will no longer be linked with an account, meaning that Cron Jobs will not complete.
